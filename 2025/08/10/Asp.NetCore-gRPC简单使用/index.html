1、安装依赖

对于 asp.net core 服务端项目安装以下即可，<font style="color:rgb(22, 22, 22);background-color:rgb(230, 230, 230);">Grpc.AspNetCore</font> 元包中包含对 <font style="color:rgb(22, 22, 22);background-color:rgb(230, 230, 230);">Grpc.Tools</font> 的引用。 服务器项目可以使用 Visual Studio 中的包管理器或通过将 <font style="color:rgb(22, 22, 22);background-color:rgb(230, 230, 230);">Grpc.AspNetCore</font> 添加到项目文件来添加 <font style="color:rgb(22, 22, 22);background-color:rgb(230, 230, 230);"><PackageReference></font>：

客户端项目应直接引用 <font style="color:rgb(22, 22, 22);background-color:rgb(230, 230, 230);">Grpc.Tools</font> 以及使用 gRPC 客户端所需的其他包。 运行时不需要工具包，因此依赖项标记为 <font style="color:rgb(22, 22, 22);background-color:rgb(230, 230, 230);">PrivateAssets="All"</font>：

2、gRPC 服务端

创建一个 Asp.Net Core 应用

添加 Protos 和 Services 文件夹，Protos 文件夹下存放 .proto 文件，Services 文件夹下存放 grpc 服务的实现

安装 Grpc.AspNetCorenuget 包

在 Protos 文件夹下创建 greet.proto 文件

在.csproj 项目文件中添加 Protobuf 引用.proto 文件，默认情况下，会为 <font style="color:rgb(22, 22, 22);background-color:rgb(230, 230, 230);">.proto</font> 项组中包含的每个 <font style="color:rgb(22, 22, 22);background-color:rgb(230, 230, 230);"><Protobuf></font> 文件都生成服务器和客户端资产。 若要确保服务器项目中仅生成服务器资产，请将 <font style="color:rgb(22, 22, 22);background-color:rgb(230, 230, 230);">GrpcServices</font> 属性设置为 <font style="color:rgb(22, 22, 22);background-color:rgb(230, 230, 230);">Server</font>。

编写 GrpcGreeter 实现业务功能

至此完成一个简单服务端代码的编写

工具包会生成表示在所包含 <font style="color:rgb(22, 22, 22);background-color:rgb(230, 230, 230);">.proto</font> 文件中定义的消息的 C# 类型。

对于服务器端资产，会生成抽象服务基类型。 基类型包含 <font style="color:rgb(22, 22, 22);background-color:rgb(230, 230, 230);">.proto</font> 文件中所含的所有 gRPC 调用的定义。 创建一个派生自此基类型并为 gRPC 调用实现逻辑的具体服务实现。 对于 <font style="color:rgb(22, 22, 22);background-color:rgb(230, 230, 230);">greet.proto</font>（前面所述的示例），会生成一个包含虚拟 <font style="color:rgb(22, 22, 22);background-color:rgb(230, 230, 230);">GreeterBase</font> 方法的抽象 <font style="color:rgb(22, 22, 22);background-color:rgb(230, 230, 230);">SayHello</font> 类型。 具体实现 <font style="color:rgb(22, 22, 22);background-color:rgb(230, 230, 230);">GreeterService</font> 会替代该方法，并实现处理 gRPC 调用的逻辑。
